## User Inputs

<Description>
  `userInputs` properties can be used to specify which word shouldn't be used
  for a password. That way you can force user to avoid using sensitive data such
  as: their email, login, name, etc.
</Description>

<Description>
  In this example try to type `thisisalongpassword`, the score should be really
  low as the word has been "banned" using `userInputs` properties.
</Description>

<Canvas>
  <Story name="User Input">
    <UncontrolledPasswordStrengthMeter
      name="user-inputs"
      estimate={zxcvbn}
      strength={[
        { color: 'red', t: 'veryWeak' },
        { color: 'orange', t: 'weak' },
        { color: 'yellow', t: 'medium' },
        { color: 'green', t: 'strong' },
        { color: 'green', t: 'veryStrong' },
      ]}
      title="Password Strength with User Inputs"
      userInputs={['thisisalongpassword']}
    />
  </Story>
</Canvas>
